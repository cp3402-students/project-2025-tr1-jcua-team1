Goal/s

The goals of the website can be broken into two main categories - client goals and development goals.

Client Goals

The client (Baizonn Learning Centre) needed a website that could showcase their services in order to engage with new families. The client goal for this website was to increase the enrolment of students of all academic backgrounds from primary school to junior college. More specifically, the website needed to:

  • clearly highlight the work done by the Baizonn Learning Centre
  • show potential new clients upcoming events and the classes being offered
  • allow interested customers to easily connect with the business.
Therefore, the goal of the website was to streamline the process of finding and registering an interest in Baizonn Learning Centre classes, encouraging users to being the process of enrolling in the program.

Development Goals

As well as producing a website that is able to satisfy the client goals there were a number of development goals that aimed to improve and guide the end solution. The website had a goal of creating a custom theme specfic to the Baizonn Learning Centre rather than utilising a standard theme available through WordPress. Additionally, the website would utilise a customiser that would allow someone with no coding experience to modify the appearance of the website to suit the clients future needs.

Local Environments

At the beginning of the project our team had two members attempt some local development in XXAMP whilst the other two members attempted this development in Docker. Each team member needed to understand the process of their local development environment, note the ease of use and be able to provide feedack as to whether it would be a useful tool for the project.

XXAMP

XXAMP was found to be very easy to setup after downloading the software. In order to run a local development environment the relevant files needed to be placed into the xampp\htdocs directory from the device drive. Once in the folder the Apache and MYSQL modules in the XXAMP Control Panel could be started and the website visited at localhost/folder_name. This process meant that XXAMP was very easy to use and run different websites at the same time.

Docker

Similar to XXAMP, Docker was found to be very easy to setup with easy integration into Visual Studio Code (the preferred code editor for our team) to create the relevant .yml and .env files. The control and updating of the website through the command line allowed the repository to be easily cloned and pulled from as required.

For the project our team decided to maintain consistency and all utilise Docker for our local development. This allowed for simplified troubleshooting and file sharing with all team members following the same processes throughout their local development. Docker was selected as the environment of choice due to the simplicity, clear steps to setup and clone repositories and the incorporation of a .env file to store passwords etc. that benefitted proejct management within the team.

AI Use Acknowledgement

I acknowledge the use of ChatGPT to assist in the development of a custom WordPress theme based on the Underscores starter theme. This generative AI was used to guide implementation of specific theme features such as header customization, front-page layout, and other theme components.

Additionally, AI was used to generate visual illustrations throughout the website. These images were created using generative AI tools with carefully written prompts that convey the desired message or emotion for each section. Image descriptions and captions include the original prompts used to generate them for transparency and attribution purposes.

For example, the following prompt was used to generate a testimonial-themed illustration:

"Generate a simple illustration with no text that encompasses 'Don't Just Take Our Word For It! Find Out What Parents and Students Think of Our Programs!'"

The following structured prompt was used to interact with ChatGPT. This prompt was designed to request context-aware, file-specific guidance for building and customizing WordPress themes:

Prompt:

I'm building a custom WordPress theme based on the Underscores starter theme, and I want help with [**].

Goal

Please help me customise this feature to match the following goal:
- [**]

Scope

  • Which WordPress template files and theme folders are involved
  • Whether any functions need to be added to functions.php
  • Any required HTML, PHP, or CSS code
  • Use of relevant WordPress functions, hooks, or filters
  • Compatibility with the Block Editor (Gutenberg) if applicable
  • Where to place any custom JavaScript or assets

Guidance & Examples

  • Walk me through the solution step-by-step
  • Provide clear code examples and explanations
  • Mention which file(s) each code block should go into
  • Recommend best practices when working with Underscores
  • Provide links to relevant WordPress documentation if available

Output Format

  1. Overview – What’s being built and why
  2. Files Affected – List of filenames and paths
  3. Code Snippets – Clearly labeled blocks of code
  4. Instructions – Where and how to implement the code
  5. Additional Notes – Caveats, improvements, or references

The outputs from these prompts were directly applied in building the theme and customising code within specific WordPress files such as header.php, functions.php, and style.css. All outputs were reviewed, tested, and modified as needed to meet project goals.
